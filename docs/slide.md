# アイデアとアプリケーション(モンスター探しゲーム 仮) 2枚
- ユーザの位置を推定し, モンスターとの距離を測る
- ユーザからはモンスターがどこにいるのかわからない + モンスターは動作し続けている
- ユーザが動いているモンスターの位置を特定, モーションによりモンスターを捕まえることができる.

## 位置測位部分　位置測位の全体像1枚
[仕様]
- 複数端末(スマホ)beaconとして電波を発する, 受信端末がraspi(ユーザが持つ一台)
- 2Dによる位置測位, 精度・どの程度の範囲・グリッド分割はXXXXXX程度を目標としている.

[利用技術]
- RSSIによる位置測位 1~2・位置推定入れるなら3枚(位置推定アルゴリズムは今後の課題)
  - raspi同士にセンサーが必要ないのがうれしいね
  - プライバシー・バッテリーにやさしい・ハードウェア的に困難でない.
  - 位置測位の受信raspi

## アプリケーション側 1枚
[仕様]
- ゲームのメイン部分を動作させるサーバによりユーザの位置を推定
- モンスターが近ければ光る. 詳細な仕様はのちに決定.(例: モーション検知した時に,近くにモンスターがいればモンスターとの距離等に対応する色で光)
- 受信端末raspiのモーション検知?, ユーザが持つrapsi二台にしてモーション検知・RSSI受信で役割分担させる?

[利用技術]
- senseHatはモーション検知 or RSSI受信用raspiのどちらかにつけて光らせる
- ジャイロセンサーによるモーション検知

# グループ内の分担 1枚
